# ⚡ Just Ask: AI-Powered Document Assistant

![Just Ask Header](https://via.placeholder.com/1200x300/0f0f23/818cf8?text=Just+Ask+-+RAG+Document+Assistant)

**Just Ask** is a production-grade Retrieval-Augmented Generation (RAG) application that transforms how users interact with their document libraries. Instead of manually searching through PDFs, users can simply upload their documents and ask natural language questions to get instant, accurate answers with citations.

---

## 🌟 Key Features

- **Isolated Document Contexts**: Each uploaded PDF has its own dedicated chat workspace. The AI strictly focuses its memory and context on the selected document, ensuring hallucination-free, highly specific answers.
- **Persistent Local Memory**: Conversations are intelligently saved to your browser's local storage per document. A rolling limit of 50 messages per document ensures lightning-fast performance without memory bloat.
- **Streaming Responses**: Get real-time, token-by-token streaming responses directly from the LLM, providing a snappy and interactive user experience.
- **Source Citations**: Every answer generated by the AI includes exact page numbers, snippets, and relevance scores tracing back to the original PDF.
- **Beautiful Glassmorphism UI**: A modern React-based frontend featuring drag-and-drop uploads, dashboard grids, hover micro-animations, and a responsive design.
- **Production Ready**: Fully configured for 1-click cloud deployment via Render, utilizing asynchronous PostgreSQL, Gunicorn, and Uvicorn.

---

## 🏗️ Tech Stack

### Frontend
- **Framework**: React 19 + Vite
- **Styling**: Modern Vanilla CSS (CSS Variables, Glassmorphism, CSS Grid/Flexbox)
- **Icons & Markdown**: Lucide React, React-Markdown
- **Routing/State**: Advanced React State Management & `localStorage` serialization

### Backend
- **Framework**: FastAPI (Async Python)
- **Vector Database**: ChromaDB (with Sentence-Transformers `all-MiniLM-L6-v2`)
- **LLM Engine**: Groq Cloud (`llama-3.3-70b-versatile`) via `httpx`
- **Database**: PostgreSQL (Production) / SQLite (Local) via async SQLAlchemy
- **Document Parsing**: PyPDF
- **Search Optimization**: Hybrid Search (Vector + BM25 keyword matching)

## 🚀 Quick Start (Local Development)

### 1. Backend Setup
```bash
cd backend
python -m venv venv

# Activate venv
# Windows:
.\venv\Scripts\activate
# Linux/Mac:
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Configure environment
cp .env.example .env
# Edit .env and add your free GROQ_API_KEY from console.groq.com

# Start the server
uvicorn app.main:app --reload
```

### 2. Frontend Setup
```bash
cd frontend/justask
npm install

# Start the Vite development server
npm run dev
```

Open `http://localhost:5173` in your browser.

## ☁️ Deployment

This repository is configured as **Infrastructure as Code** using a Render Blueprint (`render.yaml`).

1. Push this repository to GitHub.
2. Log into the [Render Dashboard](https://dashboard.render.com/).
3. Click **New +** → **Blueprint** and select your repository.
4. Render will automatically provision:
   - A PostgreSQL Database
   - A Python Web Service (Backend)
   - A Static Site (Frontend)
5. Add your `GROQ_API_KEY` to the Backend Web Service's environment variables.

For more details, see [docs/deployment.md](./docs/deployment.md).
